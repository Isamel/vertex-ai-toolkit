# Export Guide

VAIG can export analysis results and session data to JSON, Markdown, or HTML formats.

## Export Formats

| Format | Flag | Description |
|--------|------|-------------|
| Markdown | `md` or `markdown` | Structured report with headers and formatting |
| JSON | `json` | Machine-readable structured data |
| HTML | `html` | Self-contained styled HTML page |

## Exporting from `ask`

Export results directly from a single-shot query:

```bash
# Export to Markdown
vaig ask "Analyze this code for security issues" -f app.py --format md -o report.md

# Export to JSON
vaig ask "Review this config" -f nginx.conf --format json -o analysis.json

# Export to HTML
vaig ask "Threat model this service" -f architecture.yaml --format html -o report.html
```

## Exporting from `live`

Export infrastructure investigation results:

```bash
vaig live "Full health check of production" \
  --cluster prod --format md -o health-report.md

vaig live "Analyze error spike" \
  -s rca --cluster prod --format json -o rca-data.json
```

## Exporting Past Sessions

Export a previously saved session:

```bash
# Find the session ID
vaig sessions list
vaig sessions search "incident"

# Export it
vaig export abc123-def456 -f md -o incident-report.md
vaig export abc123-def456 -f json -o incident-data.json
vaig export abc123-def456 -f html -o incident-report.html

# Print to stdout
vaig export abc123-def456
```

## Export Structure

### Markdown Report

```markdown
# VAIG Analysis Report

**Model**: gemini-2.5-pro
**Skill**: rca
**Date**: 2024-03-13T14:30:00Z
**Estimated Cost**: $0.0234
**Tokens**: 15,234 total (12,100 prompt + 3,134 completion)

## Question

Why is the payment-service returning 503 errors?

## Context Files

- `k8s/payment-service.yaml`
- `logs/error.log`

## Response

[Full AI response here]

## Agent Details

### evidence_collector

[Agent 1 output]

### hypothesis_builder

[Agent 2 output]

---
*Generated by VAIG v0.1.0*
```

### JSON Export

```json
{
  "question": "Why is the payment-service returning 503 errors?",
  "response": "Based on the analysis...",
  "metadata": {
    "model": "gemini-2.5-pro",
    "skill": "rca",
    "timestamp": "2024-03-13T14:30:00Z",
    "tokens": {
      "prompt_tokens": 12100,
      "completion_tokens": 3134,
      "total_tokens": 15234
    },
    "cost": "$0.0234",
    "vaig_version": "0.1.0"
  },
  "context_files": [
    "k8s/payment-service.yaml",
    "logs/error.log"
  ],
  "agent_results": [
    {
      "agent_name": "evidence_collector",
      "content": "..."
    },
    {
      "agent_name": "hypothesis_builder",
      "content": "..."
    }
  ]
}
```

### HTML Report

The HTML export generates a self-contained page with embedded CSS. It includes:
- Styled header with metadata (model, skill, date, cost)
- Question section
- Full response rendered as preformatted text
- Footer with VAIG version

The HTML is designed to be portable — no external dependencies, works in any browser.

## Export Metadata

Every export includes:

| Field | Description |
|-------|-------------|
| `model` | The Gemini model used |
| `skill` | Active skill (if any) |
| `timestamp` | ISO 8601 timestamp |
| `tokens` | Token usage breakdown (prompt, completion, total) |
| `cost` | Estimated API cost |
| `vaig_version` | VAIG version |

## Programmatic Export

The export system can be used programmatically:

```python
from vaig.cli.export import ExportPayload, ExportMetadata, format_export

payload = ExportPayload(
    question="What is wrong with this service?",
    response="The analysis shows...",
    metadata=ExportMetadata(
        model="gemini-2.5-pro",
        skill="rca",
        timestamp="2024-03-13T14:30:00Z",
        tokens={"prompt_tokens": 1000, "completion_tokens": 500, "total_tokens": 1500},
        cost="$0.01",
        vaig_version="0.1.0",
    ),
    context_files=["app.py"],
    agent_results=[],
)

# Format as markdown
md = format_export(payload, "md")

# Format as JSON
json_str = format_export(payload, "json")

# Format as HTML
html = format_export(payload, "html")
```
