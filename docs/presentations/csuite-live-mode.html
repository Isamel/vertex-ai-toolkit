The Challenge

Manual Monitoring is Slow, Reactive, and Expertise-Gated

⏱️
Slow Diagnosis

Engineers navigate kubectl, Cloud Logging, and Cloud Monitoring simultaneously — diagnosis takes minutes or hours instead of seconds.

🔇
Reactive, Not Proactive

Teams find out about degraded services only after customers report impact. There is no systematic pre-alert health sweep.

🎓
Expertise-Gated

Effective GKE troubleshooting requires deep knowledge of Kubernetes, Istio, ArgoCD, Helm, and Cloud Monitoring APIs.

📉
High MTTR

Mean time to resolution is inflated by context-switching between 5+ tools and manual correlation of unstructured log data.

⚠️
Inconsistent Runbooks

Every engineer follows a different diagnostic path, producing inconsistent outputs and missed signals during incidents.

💸
Invisible Cost Waste

Idle containers, over-provisioned workloads, and namespace cost anomalies go undetected without dedicated analysis tooling.

The Solution

vaig live — AI-Powered
Infrastructure Investigation

A single command that deploys a fleet of autonomous AI agents against your GKE cluster and GCP stack — returning structured, actionable insights in seconds.

  • Natural language queries — no kubectl expertise required
  • 47 read-only tools across GKE, Cloud Logging, Monitoring, ArgoCD, Helm & Datadog
  • Multi-agent pipeline with parallel gatherers and AI synthesis
  • Structured HTML/JSON/Markdown reports exportable to any BI tool
  • --watch mode with change-diff tracking between iterations
  • Autonomous cluster discovery and environment health checks built in
  • Skill routing — auto-selects the right specialist agent for the query
# Investigate a degraded service
$ vaig live "Why is payment-service slow?"
--cluster prod --namespace api
# Full health check with a skill
$ vaig live "Health check staging"
--skill service-health --format html
# Continuous watch mode (every 60s)
$ vaig live "OOM kills in prod?"
--watch 60 --namespace production
# Preview without running (safe)
$ vaig live "Check pod health"
--dry-run

Key Capabilities

What vaig live Delivers

🔍
Discover & Diagnose
Autonomous cluster scanning maps every workload automatically. Real-time diagnostics inspect pods, nodes, and events — answers in seconds.
📜
Log Intelligence
Queries Cloud Logging with AI-driven filtering — surfaces OOM kills, crash loops, and 5xx patterns automatically.
📊
Metrics Correlation
Pulls Cloud Monitoring time-series and Datadog APM metrics, correlating CPU, memory, and latency anomalies.
🤖
Multi-Agent Pipeline
Parallel gatherer agents feed a synthesis agent — breadth and depth simultaneously, not sequentially.
⚙️
Skill Auto-Routing
Query scored against all registered skills — best match activates the right specialist pipeline automatically.
🔄
Watch Mode + Diff
Re-executes every N seconds and highlights what changed between iterations — catch regressions the moment they appear.
📄
Export Reports
One-command HTML SPA dashboards (including full watch session history), JSON for BI pipelines, or Markdown for incident tickets.
🛡️
Safe & Self-Healing
Designed to be safe for production: Kubernetes write tools are available but run only when explicitly invoked. A separate vaig doctor command lets you validate configuration, credentials, and cluster connectivity whenever you choose.

Live Demo

Running a vaig live Investigation

  1. 1
    Install with live support
    Adds GKE, Cloud Logging, Monitoring dependencies
  2. 2
    Configure cluster & project
    YAML config or CLI flags — takes 2 minutes
  3. 3
    Run a dry-run first
    See exactly what agents & tools would execute
  4. 4
    Execute and export
    Get an HTML report in ~30–90 seconds
  5. 5
    Enable watch for continuous monitoring
    Re-runs every N seconds — keeps the team informed
Install
$ pip install "vertex-ai-toolkit[live]"
Dry-run preview
$ vaig live "Check pod health in prod" --dry-run
Full health report
$ vaig live "Service health check" \
  --skill service-health \
  --cluster prod --namespace api \
  --format html --open
Continuous watch
$ vaig live "OOM kills & crash loops?" \
  --namespace production --watch 60
RCA with export
$ vaig live "Investigate 5xx spike at 14:30" \
  --skill rca -o incident-rca.md --format md
